**Scientific summary:** a research framework for studying how SLAM systems can adaptively fuse camera, IMU, LiDAR, and RGB-D measurements while explicitly reasoning about uncertainty, reliability, degradation, and failure risk.

> Central question: **How can a SLAM system adaptively fuse heterogeneous sensors while explicitly reasoning about uncertainty, reliability, degradation, and failure modes?**

This repository is claim-disciplined. It now includes an executable deterministic **Synthetic Demo** that generates metrics, figures, `demo.gif`, and `demo.mp4` from code. The generated numbers and media are synthetic research diagnostics, not real-world benchmark results.

## Research motivation

Visual-inertial, LiDAR-visual-inertial, and RGB-D SLAM pipelines often use fixed measurement covariances even when sensing conditions change. In field robotics, cameras lose texture or suffer motion blur, LiDAR returns become sparse, depth sensors saturate, and IMUs drift. Fixed fusion can over-trust degraded measurements and produce overconfident failures. This project makes sensor reliability an explicit state-estimation signal.

## Synthetic sensor-fusion pipeline

```text
Ground-truth trajectory + landmarks
        │
        ▼
Synthetic camera / IMU / LiDAR / RGB-D streams
        │
        ▼
Degradation: visual quality loss, LiDAR dropout, depth noise, IMU bias drift
        │
        ▼
Confidence estimation + covariance-aware pseudo-precision weighting
        │
        ▼
Mahalanobis-style rejection + adaptive fusion
        │
        ▼
Trajectory, uncertainty, metrics, figures, GIF, MP4
```

## Metrics

The Synthetic Demo exports ATE RMSE, RPE RMSE, trajectory drift, mean NEES, mean NIS, tracking failure-rate proxy, rejected-measurement count, covariance trace, log determinant, innovation norm, uncertainty score, and failure-risk proxy. These metrics are synthetic diagnostics only.

## Limitations

- The demo is a lightweight 2D synthetic simulator, not a production SLAM system.
- Real-world results on EuRoC, KITTI, TUM RGB-D, TUM-VI, or ROS2 bags remain pending until loaders, manifests, and metric files are committed.
- The failure-risk score is a diagnostic proxy, not a calibrated probability.
- Synthetic behavior validates repository mechanics and uncertainty logging, not field robustness.
