# Experiment Manifest Schema

Every experiment that supports a scientific claim should emit a manifest. The manifest is a machine-readable contract between the code, the result artifacts, and the public claim made in the README, website, or paper notes.

## Required fields

```yaml
schema_version: 1
experiment_name: synthetic_nominal
claim_status: synthetic_only
repository:
  remote: https://github.com/panagiotagrosdouli/Adaptive-Multi-Modal-SLAM-with-Uncertainty-Aware-Sensor-Fusion
  commit: <git-sha>
  branch: <branch-name>
runtime:
  python: "3.10"
  platform: <platform-string>
  started_at_utc: <iso-8601>
  finished_at_utc: <iso-8601>
reproducibility:
  config_path: configs/example_experiment.yaml
  random_seed: 7
  deterministic: true
dataset:
  name: synthetic
  sequence: nominal_v1
  official_url: null
  redistributed: false
  notes: synthetic data generated by repository code
sensors:
  camera:
    enabled: true
    status: synthetic
  imu:
    enabled: true
    status: synthetic
  lidar:
    enabled: false
    status: planned
  rgbd:
    enabled: false
    status: planned
baselines:
  - name: fixed_weight_fusion
    status: pending_or_implemented
  - name: adaptive_uncertainty_fusion
    status: pending_or_implemented
metrics:
  ate_rmse_m: null
  rpe_rmse_m: null
  nees_mean: null
  nis_mean: null
  tracking_failure_rate: null
  runtime_fps: null
artifacts:
  metrics_json: results/synthetic_nominal/metrics.json
  trajectory_plot: results/synthetic_nominal/figures/trajectory.png
  reliability_plot: results/synthetic_nominal/figures/reliability.png
  fusion_weights_plot: results/synthetic_nominal/figures/fusion_weights.png
limitations:
  - synthetic run only
  - no real dataset benchmark claim
  - risk score is diagnostic and uncalibrated
```

## Claim statuses

| Value | Meaning |
| --- | --- |
| `synthetic_only` | Supports a synthetic sanity-check claim only. |
| `real_dataset_pending` | Loader/config may exist, but no reportable result is available. |
| `real_dataset_reportable` | Result is generated from committed code and can be cited in repository docs. |
| `ablation_reportable` | Comparison is generated from committed configs and metrics. |
| `diagnostic_only` | Artifact is useful for debugging but not a benchmark. |

## Scientific motivation

SLAM experiments are easy to overstate because small changes in timestamp association, calibration, alignment, outlier rejection, or random seed can materially change the reported ATE/RPE/NEES/NIS. A manifest makes these assumptions inspectable and supports reproducibility.

## Engineering motivation

A manifest gives CI and reviewers a simple object to validate. It also allows website pages, paper tables, and README snippets to consume generated artifacts instead of manually copied claims.

## Validation checklist

- [ ] All required keys exist.
- [ ] `repository.commit` is a concrete SHA.
- [ ] `reproducibility.random_seed` is set for synthetic experiments.
- [ ] Dataset redistribution status is explicit.
- [ ] Metrics are either numeric or explicitly `null` with Pending status.
- [ ] Limitations are present.
- [ ] Every figure path exists before a claim is published.
