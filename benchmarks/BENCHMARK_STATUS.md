# Benchmark Status Matrix

This file prevents accidental benchmark inflation. A benchmark row is reportable only when the repository contains the corresponding loader or adapter, config, metric artifact, result manifest, and reproduction command.

## Baselines

| Baseline | Status | Required before reporting numbers |
| --- | --- | --- |
| Visual-only odometry | Prototype / Pending | frontend tracking path, config, ATE/RPE output, failure-rate logging |
| IMU-only dead reckoning | Prototype / Pending | IMU propagation, bias/noise model, drift metrics, synthetic validation |
| LiDAR-only odometry | Planned | LiDAR feature extraction or scan matching, dataset adapter, metric output |
| Fixed-weight sensor fusion | Prototype / Pending | deterministic fusion config, metrics, comparison against adaptive policy |
| Adaptive uncertainty-aware fusion | Implemented baseline components / Pending benchmark | complete experiment harness and result manifests |
| Oracle sensor reliability | Planned scaffold | ground-truth degradation schedule or synthetic reliability labels |

## Dataset targets

| Dataset | Status | Intended use | Reporting rule |
| --- | --- | --- | --- |
| Synthetic nominal | Implemented / evolving | deterministic sanity checks | May report only generated synthetic metrics. |
| Synthetic visual degradation | Prototype / Pending | test adaptive visual reliability | Needs degradation schedule and metric JSON. |
| Synthetic IMU noise | Prototype / Pending | test covariance/risk response | Needs noise schedule and NEES/NIS logs. |
| Synthetic LiDAR dropout | Prototype / Pending | test modality dropout handling | Needs dropout schedule and weights plot. |
| Synthetic depth noise | Prototype / Pending | test RGB-D reliability scaffold | Needs depth-noise model and metrics. |

| TUM RGB-D | Planned / Pending | RGB-D SLAM evaluation | Do not report numbers until association and depth scaling are validated. |
| TUM-VI | Planned | visual-inertial stress testing | Documentation and loader required. |
| nuScenes | Planned scaffold | autonomous-driving multi-modal extension | Large-scale adapter and licensing documentation required. |

## Minimum manifest fields

```yaml
experiment_name: synthetic_visual_degradation
status: synthetic
repository_commit: <git-sha>
config_path: configs/synthetic_visual_degradation.yaml
random_seed: 7
dataset:
  name: synthetic
  sequence: visual_degradation_v1
  redistributed: false
baselines:
  - fixed_weight_fusion
  - adaptive_uncertainty_fusion
metrics:
  ate_rmse_m: results/.../metrics.json
  rpe_rmse_m: results/.../metrics.json
  nees_mean: results/.../metrics.json
  nis_mean: results/.../metrics.json
figures:
  - results/.../figures/trajectory.png
  - results/.../figures/fusion_weights.png
limitations:
  - synthetic data only
  - diagnostic risk score is not calibrated probability
```

## Benchmark reporting checklist

Before adding a number to README, website, paper notes, or a benchmark table:

- [ ] The metric was generated by committed code.
- [ ] The config is committed.
- [ ] The random seed is fixed or the run reports seed aggregation.
- [ ] Dataset sequence and association assumptions are documented.
- [ ] Result JSON is committed or attached as an artifact.
- [ ] Figure/table generation is reproducible.
- [ ] Limitations are stated next to the result.
